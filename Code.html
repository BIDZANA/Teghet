Bienvenue sur ma page Git

Nous allons tout au long du cours remplir cette page de commandes git


Partie 1


Git add

git add [nom_du_fichier] : ajoute le fichier mentionné

git add . : Ajoute tout les nouveaux fichiers en dessous du répertoire courant dans l'arborescence

git -A : Ajoute tous les nouveaux fichiers du répertoire initial

Git commit

git commit -m "message" : permet de créer une sauvegarde du répertoire

Il est conseillé de faire des commit d'un fichier qui compile et de toujours mettre les features et leurs tests dans le même commit

Git log

git log : Affiche l'historique des commits

Git alias

Cette commande permet de créer des alias pour d'autres comamandes

Cet exemple nous montre la création d'un alias tree pour une commande de log

Git checkout

git checkout [tag] : cette commande nous rennvoie sur le commit ou la branche taguée

git checkout -b [nom_de_la_branche]: permet de se déplacer sur une branche en la créant en même temps si elle n'existe pas

Git ignore

.gitingnore : est un fichier qu'on crée pour spécifier les fichiers qu'on ne souhaite pas partager


Partie 2


Git merge

git merge --no-ff feature : Lorsqu'on exécute cette commande étant positionné sur la branche main ou master, le contenu de la branche feature est ajouté à la banche main

Git branch delete

git branch --delete feature : Supprime la branche feature

Git rebase

git rebase master: Lorsqu'on l'exécute étant sur une branche, cette commande nous permet d'apporter toutes les modifications de la branche master sur cette branche et de considérer cette branche comme la branche principale

Merge fast forward

git merge --ff-only feature : On l'exécute depuis master pour que master pointe également sur la nouvelle branche principale

Git stash

git stash : permet de mettre de côté un travail qu'on a commencé à faire sur une branche mais qu'on désire mettre de côté pour une raison ou une autre

git pop : permet de récupérer le travail mis de côté

git drop : permet de supprimer le travail mis de coté

Git revert

git revert HEAD~2 : permet d'annuler les modifications d'un commit en créant un nouveau commit qui ne contient pas ces modifications
